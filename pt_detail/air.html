PROJECT DETAILS

목표

INAVI AIR

아이나비 에어 네비게이션 APP을 유지보수하여 사용자에게 정확한 도로 교통 정보 및 길안내를 제공

기간 : 2023.07 ~ 2024.04
역할 : Agent 서버 개발
기술 스택 : SpringBoot 2, Java 11, Mysql 8

과정

INAVI APP Navigation Service

Spring Boot 2.2 기반,
일일 UV 3만, 월별 UV 15만을 기록하는
네비게이션 APP 서비스 유지보수를 담당

1. 서비스 기능 구현

  • HTTP/TCP 엔진 통신 기반 네비게이션 AGENT 서버 운영
  • 다양한 캐시 지급 API 구현
    • 난수 기반 쿠폰 지급 시스템,SK planet과 협력하여 구글 AD 광고 캐시 지급 API 개발
    • 협업 과정 : 검증/지급에 대한 순서도를 작성하여 구성원들에게 공유하여 검토 후 개발 진행
    • 이력 관리 : 사용자에게 캐시 지급시에 발생한 히스토리를 RDBMS에 저장
  • 사용자 제보 서비스 구현
    • 도로 교통 문제를 제보하면 캐시를 받는 기능
    • 사용자가 제보한 내용을 관리자가 확인하여 완료 처리하면, 제보하기 API에서 사용자를 검증 하고 캐시 지급

2. 이슈 관리

  • 주간 스크럼 앱 리뷰 분석을 통해 사용자 의견을 수집, 분석하여 서비스 개선에 적극 반영
  • 통신 중단 시 Slack 알림을 통해 신속히 이슈를 감지하고 대응, 서비스 안정성 강화
  • Jira,Slack등으로 문의가 들어오는 이슈를 신속하게 로그 기반으로 식별하고 문제 해결
  • 엔진 및 단말기 관련 이슈를 담당자들과 협력하여 문제 해결

3. 자동화 시스템 운영

  • 캐시 통계 자동 추출
    • Shell script를 활용, Slave DB에서 파라미터에 따라 데이터를 조회 후 메일 발송
  • 캐시 상품 관리
    • Admin 페이지에 캐시 상품 관리 기능 추가로 사업팀에서 직접 상품 CRUD 작업을 수행
  • Gradle을 이용한 빌드 및 배포 자동화로 배포 시간 70% 이상 단축,
    • Gradle을 이용한 빌드 및 배포 자동화
    • systemctl을 사용한 서비스 자동 재기동으로 시스템 가동률 99.9% 유지
  • 서비스 중단 없는 배포 실현
    • L4 스위치를 활용하여 작업 중인 서버의 트래픽을 중지시켜 사용자에게 서비스 중단이 발생하지 않도록 조치

4. 문제 해결

  • 통계 쿼리속도 개선
    • 쿼리 속도를 90%이상 개선하기 위해 인덱스를 활용하여 최적화
    • 서비스의 안정성을 유지하기 위해 Slave DB에서 진행
  • 엔진 통신 문제 해결
    • 문제 발생 시 서버에서 로그를 기반으로 선 검토하고, 엔진 문제로 확인될 경우
      요청 시간, 통신 방식, 파라미터 정보를 정리하여 엔진 담당자에게 문의
    • 다양한 통신 문제를 담당자와 협력하여 해결

성과

팀원들과 협력하여 각자의 전문성을 활용, 이슈와 문제를 효과적으로 해결하고 서비스 품질을 향상시켰습니다.
