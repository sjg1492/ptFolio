PROJECT DETAILS

목표

THINKWARE CONNECTED

블랙박스와 통신하여 차량 정보를 실시간으로 모니터하며, 이벤트 발생 시 사용자에게 알림,정보를 제공합니다.

기간 : 2023.02 ~ 2024.04
역할 : Agent 서버 관리자 및 개발자
기술 스택 : SpringBoot 2, Java 11, Mysql8, Nginx

과정

Connected Blackbox Service

Spring Boot 2.2 기반, 아시아, 유럽, 캐나다,
아메리카 42개국의 1만 4천명 사용자 대상
블랙박스 커넥티드 서비스 Agent 서버 운영

1. 서비스 기능 구현

  • 블랙박스 MQTT, HTTP 통신 기반 서비스 운영
  • 통계 서비스 구축 작업을 위한 AOP 기능 추가
    • AOP를 활용하여 단말기별, 국가별, 월별 사용자 행동 및 데이터 분석 서비스 개발
    • 데이터 수집 로직 분리를 통해 사업부서가 월별 UV, PV 데이터를 효율적으로 확인할 수 있도록 구현
  • 일본 커넥티드 APP 출시에 따른 이슈 검토 및 추가 개발
    • 일본 APP 출시에 발생할 이슈를 검토하고 대응
  • WI-FI 블랙박스 단말기 고온으로 인한 전원 꺼짐 현상 이슈 대응
    • 로그 API 개발팀과 협업하여 발생 가능한 사이드 이팩트 방어 로직 작성
  • 차량 관리 서비스 구축
    • 사용자 차량 정보를 관리하기 위한 조회 서비스 REST API 기반 구축
    • 기능: 차량 이동경로 조회, 차량 마지막 주차위치 조회, 충격 이벤트 발생 위치 조회 등
  • 신규 단말기 정보 엑셀 파일 전송 검증 자동화
    • 기존 방식: 서비스 품질관리팀이 파일을 업로드 후 검증 요청, 서버 담당자가 수동 검증하여 회신
    • 개선 방식: 파일 검증 과정을 API로 자동화하여 "불필요한 메일 교환" 제거

2. 이슈 관리

  • 주간 스크럼 앱 리뷰 분석을 통해 사용자 의견을 수집, 분석하여 서비스 개선에 적극 반영
  • 통신 중단 시 Slack 알림을 통해 신속히 이슈를 감지하고 대응, 서비스 안정성 강화
  • Mantis,Slack,Jira 등으로 문의가 들어오는 이슈를 로그 기반으로 식별하고 문제 해결
  • 블랙박스 해외 영업소에서 메일로 들어오는 이슈 분석하고 문제 해결

3. 자동 배포 시스템 운영

  • Gradle을 이용한 빌드 및 배포 자동화로 배포 시간 70% 이상 단축,
    • Gradle을 이용한 빌드 및 배포 자동화
    • systemctl을 사용한 서비스 자동 재기동으로 시스템 가동률 99.9% 유지
  • 서비스 중단 없는 배포 실현
    • L4 스위치를 활용하여 작업 중인 서버의 트래픽을 중지시켜 사용자에게 서비스 중단이 발생하지 않도록 조치

4. 문제 해결

  • 통계 쿼리속도 개선
    • 쿼리 속도를 90%이상 개선하기 위해 인덱스를 활용하여 최적화
  • Multipart 통신 에러
    • 파일 데이터 전송 중 Multipart 방식과 Chunk 방식 간의 불일치 문제 발생
    • 데이터 전송 방식을 Chunk 방식으로 조정, Chunk 크기를 명시적으로 설정하여 문제 해결

성과

팀원들과 협력하여 각자의 전문성을 활용, 이슈와 문제를 효과적으로 해결하고 서비스 품질을 향상시켰습니다.

1.서버 운영 최적화

서버 가동률 99.9% 유지

2. 이슈 해결률 100% 달성

Mantis, Teams 활용 이슈 식별 및 해결 프로세스 개선

3.신속한 서버 문제 해결

로그 기반 서버/클라이언트 문제 진단 및 해결
